G5 진행 방식

1) 트라이얼 준비
	•	스테이지 규칙에 맞춰 3개의 선택지를 구성합니다.
	•	BroadFamily: 같은 악기군 2개 + 다른 악기군 1개
	•	WindsSplit: 목관 2개 + 금관 1개 또는 반대로 구성
	•	SimilarWithinGroup: 같은 유사그룹에서 2개 동일, 1개는 같은 그룹의 다른 악기
	•	CultureSplit: 전통악기 2개 + 서양악기 1개 또는 반대로 구성
	•	각 선택지에는 InstrumentEntry(표시명, 분류, 샘플)에서 무작위로 AudioClip을 하나 할당합니다.
	•	3개 옵션을 셔플해 배치하고, 그중 “다른 소리”의 인덱스를 정답으로 기록합니다.

2) 프리뷰 단계
	•	입력 비활성화 상태에서 A→B→C 순서로 각 선택지를 하이라이트하며 1회씩 재생합니다.
	•	샘플 1개당 재생시간은 실제 길이를 따르되 최대 1.5초로 제한, 옵션 사이 간격은 interOptionDelay로 제어합니다.
	•	프리뷰가 끝나면 gapAfterPreview만큼 무음 대기합니다.

3) 응답 단계
	•	버튼 입력이 활성화되고, 사용자는 A/B/C 중 “다른 소리”를 고릅니다.
	•	반응시간은 입력 허용 시점부터 정답을 클릭한 시점까지를 측정합니다(정답일 때만 평균에 반영).
	•	필요시 Replay All 버튼으로 프리뷰를 다시 들을 수 있습니다(입력 허용 중에만).

4) 피드백/기록
	•	정답/오답을 자막으로 보여주고, 정답인 선택지의 소리를 한 번 더 들려줄 수 있습니다.
	•	내부적으로 트라이얼 수, 정답 수, 정답 반응시간 합계를 누적합니다.

5) 다음 트라이얼로
	•	짧은 텀(약 0.4초) 후 다음 트라이얼을 생성해 1)~4) 과정을 반복합니다.
	•	totalTrials회 진행 후 게임 종료 콜백을 통해 (total, correct, avgReaction)를 GameHostController에 전달합니다.

6) 결과/클리어 판정
	•	GameHostController가 정확도 correct / total를 기준으로 스테이지 클리어 여부를 판정하고 결과 오버레이를 띄웁니다.
	•	평균 반응시간은 정답 트라이얼들만 산출값에 포함됩니다.

화면/조작 포인트
	•	버튼: A/B/C 3개 + 프리뷰 다시듣기 버튼
	•	라벨 하이라이트: 프리뷰 시 해당 선택지 라벨이 살짝 확대되는 효과
	•	자막: 진행 안내, 정답/오답 피드백 등은 ServiceHub.Caption으로 표출
	•	일시정지: 설정 모달을 열면 Pause()로 입력/오디오가 모두 멈추고, 닫으면 Resume()로 복구
	•	시간 기준: useUnscaledTime 기본값으로 타임스케일 영향을 받지 않습니다.

인스펙터/데이터 세팅 요약
	•	InstrumentEntry 배열: id/표시명/Family/Culture + AudioClip[]을 등록
	•	SimilarGroup 리스트: 유사 악기 변별이 필요한 그룹을 구성(예: 바이올린/비올라/첼로/더블베이스)
	•	타이밍 파라미터:
	•	interOptionDelay: A→B→C 사이 간격
	•	gapAfterPreview: 프리뷰 종료 후 입력 허용까지의 무음 대기
	•	트라이얼 수: totalTrials
	•	스테이지 규칙 매핑은 GameHostController에서 캠페인/레벨에 따라 자동 지정됩니다.

난이도/밸런스 조정 팁
	•	BroadFamily → WindsSplit → SimilarWithinGroup 순으로 점차 어려워집니다.
	•	프리뷰 구분감이 부족하면 interOptionDelay를 조금 늘리고, 샘플 길이가 너무 길면 소스 파일을 짧게 편집하거나 현재 제한(최대 1.5초)을 더 낮춰도 됩니다.
	•	데이터 풀이 충분하지 않으면 조합 생성에 실패할 수 있으니(콘솔 경고), 각 분류/그룹에 최소 2개 이상의 악기와 샘플을 넣어주세요.
